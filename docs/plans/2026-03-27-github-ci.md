# GitHub CI Workflow Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a minimal GitHub Actions workflow that runs tests, lint, and build on pull requests to `main` and pushes to `main`.

**Architecture:** Keep CI intentionally small: one workflow, one job, one Node version, and the same command sequence developers already use locally. Add a lightweight repository test that validates the workflow file contents so future edits do not silently remove required checks or triggers.

**Tech Stack:** GitHub Actions, YAML, Node.js, npm, Vitest.

---

### Task 1: Add A Failing Repository Test For The Workflow

**Files:**
- Create: `tests/github-ci-workflow.test.ts`

**Step 1: Write the failing test**

Create `tests/github-ci-workflow.test.ts`:

```ts
import { readFileSync } from "node:fs";
import { resolve } from "node:path";

describe("GitHub CI workflow", () => {
  it("defines the expected triggers and verification commands", () => {
    const workflow = readFileSync(
      resolve(process.cwd(), ".github/workflows/ci.yml"),
      "utf8",
    );

    expect(workflow).toContain("pull_request:");
    expect(workflow).toContain("push:");
    expect(workflow).toContain("branches: [main]");
    expect(workflow).toContain("npm ci");
    expect(workflow).toContain("npm run test -- --run");
    expect(workflow).toContain("npm run lint");
    expect(workflow).toContain("npm run build");
  });
});
```

**Step 2: Run the test to verify it fails**

Run:

```bash
npm run test -- --run tests/github-ci-workflow.test.ts
```

Expected: fail because `.github/workflows/ci.yml` does not exist yet.

**Step 3: Commit**

```bash
git add tests/github-ci-workflow.test.ts
git commit -m "test: cover github ci workflow"
```

### Task 2: Add The Workflow File

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Use the failing test as the contract**

Keep the test from Task 1 red until the workflow file exists with the required triggers and commands.

**Step 2: Run the focused test to verify failure**

Run:

```bash
npm run test -- --run tests/github-ci-workflow.test.ts
```

Expected: fail with file-not-found or missing-content output.

**Step 3: Write the minimal implementation**

Create `.github/workflows/ci.yml` with:

- workflow name such as `CI`
- triggers for:
  - `pull_request` on `main`
  - `push` on `main`
- one `build` job on `ubuntu-latest`
- `actions/checkout@v4`
- `actions/setup-node@v4` with a pinned Node version
- `npm ci`
- `npm run test -- --run`
- `npm run lint`
- `npm run build`

**Step 4: Run the focused test to verify it passes**

Run:

```bash
npm run test -- --run tests/github-ci-workflow.test.ts
```

Expected: pass.

**Step 5: Commit**

```bash
git add .github/workflows/ci.yml tests/github-ci-workflow.test.ts
git commit -m "ci: add github actions checks"
```
